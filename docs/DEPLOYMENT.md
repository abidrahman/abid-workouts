# Deployment

This app is configured as a static site for GitHub Pages. The production build is generated by a dependency-free Node.js script in [`scripts/build.mjs`](../scripts/build.mjs).

## Target URL

```text
https://abidrahman.github.io/abid-workouts/
```

The app uses relative asset URLs so it works locally and when served from the `/abid-workouts/` project-site subpath.

## GitHub Pages setup

This repository includes `.github/workflows/deploy-pages.yml`, which builds and deploys `dist/` from the `main` branch.

Expected repository settings:

1. Open the repository on GitHub.
2. Go to **Settings → Pages**.
3. Under **Build and deployment**, choose **GitHub Actions** as the source.
4. Push or merge into `main` to trigger the deployment workflow.

## Private repository caveat

GitHub Pages support for private repositories depends on account and repository settings. If GitHub does not allow Pages for this private repo, use one of these options:

- Make this repo public.
- Keep the source private and deploy the built `dist/` output to a public Pages repo.
- Use another no-cost static host that supports private source repositories.
